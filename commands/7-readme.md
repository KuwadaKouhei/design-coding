> **STEP7 / 全7フェーズ（最終）**　前提: 実装・テスト・レビュー完了＋`docs/` 一式　｜　現在地確認: `/status`

担当ペルソナは `readme-author`（README 整備担当）。`Agent` ツールでこのサブエージェントに
委譲するか、その役割を引き受けて `readme-generation` スキルを起動する。

次の手順で進める:

1. `docs/`（REQUIREMENTS / philosophy/* / TECH_STACK / FEASIBILITY / DESIGN / DATABASE 等）・`CLAUDE.md`・
   マニフェスト（package.json 等）・`.env.example`・実ディレクトリを読む（**`.env` は読まない**）
2. 採用技術を shields.io バッジで起こす（バージョンはマニフェストから。でっち上げない）
3. プロジェクトルートに `README.md` を生成/更新する（既存があれば手書き内容を保持してマージ）。必ず8要素を含める:
   - 技術スタック（shields バッジ）
   - プロジェクト概要
   - プロジェクト思想（発足理由・工夫点）
   - 必要な環境変数（秘密値なし）とコマンド一覧
   - ディレクトリ構成
   - 開発環境の構築方法（再現可能な手順）
   - トラブルシューティング
   - ドキュメントの説明とリンク（実在ファイルのみ）
4. 人間のレビューを受ける
